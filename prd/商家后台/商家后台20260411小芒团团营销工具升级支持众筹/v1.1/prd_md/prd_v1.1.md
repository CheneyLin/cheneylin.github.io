## 一、需求背景

### 1.1 现状问题

当前小芒团团营销工具仅支持**单一团购玩法**，无法满足商家对众筹营销场景的需求。众筹玩法的核心差异在于：
- 需要设定目标（金额/件数），达标后商品才会生产/发货
- 未达标需触发退款流程
- 需要展示众筹进度、氛围等专属信息

### 1.2 v1.1 优化背景

v1.0 版本已实现众筹玩法的基础功能，v1.1 版本针对**众筹详情页的价格条和进度条**进行阶段性细节优化，提升用户信息获取效率和视觉体验。

## 五、详细方案

### 5.1 运营后台

#### 5.1.1 小芒团团活动列表

**功能标识**: `admin_activity_list`

**前置条件**
- 商家已开通小芒团团营销工具权限
- 商家已登录商家后台

**核心逻辑**
1. 列表展示商家已创建的所有小芒团团活动
2. 列表字段包含：活动名称、玩法类型（单品团购/众筹）、截止时间、当前进度、状态、操作
3. 提供操作按钮：**创建活动**、**编辑**、**氛围配置**、**下线**
4. 玩法类型标签区分：
   - 单品团购：绿色标签
   - 众筹：蓝色标签

---

#### 5.1.2 基本配置项

**功能标识**: `admin_activity_list_popup_info`

**前置条件**
- 商家点击"创建活动"或"编辑"按钮

**核心逻辑**
- 在团购配置弹窗中展示基本配置项

**配置项清单**

| 配置项 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| 活动名称 | 文本输入 | 是 | 空 | 活动名称，用于列表展示 |
| 开始时间 | 日期时间选择器 | 是 | 当前时间 | 活动开始时间 |
| 截止时间 | 日期时间选择器 | 是 | 当前时间 + 7天 | 活动截止时间 |
| 众筹目标类型 | 下拉选择 | 是（众筹玩法） | 按金额 | 按金额 / 按件数 |
| 众筹目标值 | 数字输入 | 是（众筹玩法） | 空 | 目标金额或件数 |

---

#### 5.1.3 氛围配置项

**功能标识**: `admin_activity_list_popup_extra`

**前置条件**
- 商家在团购配置弹窗中选择"众筹"玩法

**核心逻辑**
- 选择"众筹"玩法后，展开氛围配置区

**配置项清单**

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 分享卡片图片 | 图片上传 | 系统默认图片 | 众筹活动分享时的封面图片 |
| 展示渠道 | 多选 | APP、M站、小程序（全选） | 选择众筹活动展示的渠道 |
| 达标即发货 | 开关 | 关闭 | 开启后，达标后立即安排发货（无需等待截止） |
| 时间样式 | 下拉选择 | `MM-DD HH:mm - MM-DD HH:mm` | 自定义时间展示格式 |
| 时间前缀 | 文本输入 | 空 | 如"距结束"、"剩余"等 |
| 时间后缀 | 文本输入 | `结束` | 时间后的补充说明（可选） |
| 价格名 | 文本输入 | `众筹价` | 如"众筹价"、"活动价"等 |
| 进度条样式 | 单选 | `B: 众筹` | A: 团购样式 / B: 众筹样式 |
| 进度条前缀 | 文本输入 | `已筹` | 如"已筹"、"已完成"等 |
| 进度条说明 | 文本输入 | 根据团购策略生成 | 动态生成进度说明文案 |
| 活动截止类型 | 单选 | `C: 截止后根据目标判定` | A/B/C 三种类型 |

#### 5.2.2 进度条（众筹信息区）

**功能标识**: `app_goods_details_progress_bar`

**布局结构**

```
┌─────────────────────────────────────────────────────────────┐
│ 左侧：目标                      右侧：活动规则（文字链接）         │
│ 目标：500件 / ¥20,000                                    │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│ 左侧上部                      右侧上部                       │
│ 已筹 256件                     51%                           │
│                                                             │
│ ┌─────────────────────────────────────────────────────────┐ │
│ │████████████████░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░│ │
│ └─────────────────────────────────────────────────────────┘ │
│         ↑ 进度标识小图标                                      │
│                                                             │
│ 左侧下部                      右侧下部                       │
│ 未达成目标后七天内退款           ✅成功 / ❌失败               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**字段说明**

| 区域 | 字段 | 说明 |
|------|------|------|
| 标题区左侧 | 目标 | 件数或金额目标，根据商家配置的目标类型展示 |
| 标题区右侧 | 活动规则 | 文字链接，点击查看详细规则 |
| 主体区左侧上部 | 已筹 | 已筹xx件/元 |
| 主体区右侧上部 | 进度百分比 | XX% |
| 主体区中间 | 进度条 | 带当前进度标识小图标 |
| 主体区左侧下部 | 活动规则说明 | 未达成目标后七天内退款 |
| 主体区右侧下部 | 成功/失败标识 | ✅成功 / ❌失败 |

**样式规则**

| 状态 | 进度条颜色 | 进度标识 | 说明 |
|------|------------|----------|------|
| 进行中 | 橙色渐变进度条 | 显示当前位置 | 正常状态 |
| 成功 | 绿色进度条（或不变） | 100%位置 | 显示✅成功标识 |
| 失败 | 灰色进度条 | 显示最终百分比 | 显示❌失败标识 |

**进度超额完成规则**
- 进度条最大显示100%，不允许超出容器
- 文案显示具体件数/金额（如"已筹 1200件 / 目标 100件"）
- 大额数字缩略显示（如"100万"）

- 商家开启"达标即发货"功能

**核心逻辑**
1. 价格条：与进行中状态完全一致
2. 进度条：与进行中状态完全一致，可显示超额进度
3. 按钮状态：**去支持**

**展示内容**

| 字段 | 内容 |
|------|------|
| 价格 | ¥128（众筹价） |
| 原价 | 原价 ¥199（划线） |
| 时间信息 | 距结束：3天12时（或"继续售卖中"） |
| 进度 | 120%（已筹 600件 / 目标 500件） |
| 进度条 | 橙色进度条（显示120%，但进度条按100%填充） |
| 规则说明 | 未达成目标后七天内退款 |
| 按钮 | 去支持 |

#### 5.2.8 众筹信息展示区 - 已结束状态

**功能标识**: `app_goods_details_status_end`

**前置条件**
- 商品为众筹类型
- 活动截止类型为 A（截止后活动下线）或 B（截止后继续展示但不可售）

**核心逻辑**

| 类型 | 展示策略 |
|------|----------|
| A（截止后活动下线） | 商品详情页不展示众筹信息区，展示"活动已结束"占位，底部操作按钮隐藏或变更为"查看其他活动" |
| B（截止后继续展示但不可售） | 展示众筹信息区（进度条、目标、已购人数/金额），进度条展示为灰色，底部操作按钮变更为"活动已结束" |
