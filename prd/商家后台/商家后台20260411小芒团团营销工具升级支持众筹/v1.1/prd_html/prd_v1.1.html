2. 需求背景与目标

2.1 现状问题与痛点

当前小芒团团营销工具仅支持单一团购玩法，无法满足商家对众筹营销场景的需求。众筹玩法的核心差异在于：

  1. 需要设定目标（金额/件数），达标后商品才会生产/发货
  2. 未达标需触发退款流程
  3. 需要展示众筹进度、氛围等专属信息

2.2 v1.1 优化背景

v1.0 版本已实现众筹玩法的基础功能，v1.1 版本针对众筹详情页的价格条和进度条进行阶段性细节优化，提升用户信息获取效率和视觉体验。

2.3 业务目标

目标类型描述衡量指标目标值
业务目标商家可在小芒团团营销工具中配置众筹玩法众筹活动创建成功率≥95%
用户体验App端展示众筹进度、目标、已购人数等信息众筹商品详情页转化率提升 10%
技术目标实现实时状态切换逻辑状态切换延迟≤500ms
视觉体验优化价格条和进度条阶段性展示用户满意度评分≥4.5

3. 用户与使用场景

3.1 核心用户

  • 商家运营人员：负责在商家后台配置营销活动
  • App 消费者：浏览商品并参与众筹

3.2 典型场景

  1. 商家在后台创建众筹活动，配置目标、时间、氛围样式
  2. 用户浏览 App 众筹商品详情，了解进度和规则
  3. 用户参与众筹支付，等待活动结果
  4. 活动截止后，系统判定成功/失败，触发相应流程

5.5 系统功能

5.5.1 实时状态切换

功能描述：活动截止时间到达时，系统实时触发状态判定

核心逻辑
if (当前进度 >= 目标) {
  状态变更为 "众筹成功"
  商品变为可售状态
  订单正常流转
} else {
  状态变更为 "众筹失败"
  触发退款流程
}

5.5.2 未达标退款逻辑

功能描述：众筹失败后，系统自动触发退款

核心逻辑
  • 众筹失败后，系统自动触发退款
  • 退款路径：原路退回用户支付账户
  • 退款时效：T+1 个工作日内到账
  • 通知推送：向参与用户推送"众筹失败，已发起退款"通知
  • 订单状态：变更为"已关闭-众筹未达标"

7. 异常与边界处理

异常场景处理逻辑
进度超额完成进度条按100%显示，文案显示具体件数/金额，大额数字缩略显示
数据位数过长缩略显示（如"100万"）
活动开始前点击按钮按钮不可点击，提示"活动还未开始"
网络中断（商家提交配置时）提示"网络异常，请重试"，保留已填配置数据
网络中断（用户支付时）支付状态查询，未完成则取消订单，已完成则保留
空数据状态（App 端无众筹商品）展示"暂无众筹活动"空状态页，推荐普通团购商品
权限不足（商家无小芒团团权限）提示"您暂无权限使用此功能，请联系客服开通"
库存不足（用户参与时）提示"库存不足"，禁止参与
