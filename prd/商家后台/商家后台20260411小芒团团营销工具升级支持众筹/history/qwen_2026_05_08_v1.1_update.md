# 项目修改历史

## 2026-05-08 - v1.1 迭代更新

| 修改类型 | 功能标识 | 修改说明 |
|----------|----------|----------|
| 新增 | app_goods_details_price_bar | 新增 App 端价格条组件，包含价格、价格名、原价、时间信息 |
| 新增 | app_goods_details_progress_bar | 新增 App 端进度条组件，包含目标、已筹、进度百分比 |
| 新增 | app_goods_details_status_before | 新增活动开始前状态，按钮显示"待开启" |
| 优化 | app_goods_details_status_running | 优化进行中状态，按钮显示"去支持" |
| 新增 | app_goods_details_status_success | 新增活动成功结束状态，显示✅成功标识，按钮变灰"已成功" |
| 新增 | app_goods_details_status_fail | 新增活动失败结束状态，显示❌失败标识，进度条变灰，按钮"已结束" |
| 新增 | app_goods_details_status_continue | 新增达标后继续售卖状态，按钮显示"去支持"，其他同进行中 |
| 优化 | 全局 | 优化价格条和进度条阶段性细节展示，区分5个阶段差异化表现 |

### 5个阶段详细说明

| 阶段 | 价格条 | 进度条 | 按钮状态 |
|------|--------|--------|----------|
| 活动开始前 | 正常显示 | 正常显示 | 待开启（不可点击） |
| 活动进行中 | 正常显示 | 正常显示 | 去支持 |
| 活动成功结束 | 价格保持 | 显示100%，✅成功标识 | 已成功（变灰） |
| 活动失败结束 | 价格保持 | 显示最终百分比，❌失败标识，进度条变灰 | 已结束（变灰） |
| 达标后继续售卖 | 价格保持 | 可显示超额（如120%），进度条100%填充 | 去支持 |

### 边界处理规则

| 场景 | 处理规则 |
|------|----------|
| 进度超额完成 | 进度条按100%显示，文案显示具体件数/金额 |
| 数据位数过长 | 缩略显示（如"100万"） |
| 活动开始前点击按钮 | 按钮不可点击 |
| 成功/失败标识 | 点击无详情展开 |
