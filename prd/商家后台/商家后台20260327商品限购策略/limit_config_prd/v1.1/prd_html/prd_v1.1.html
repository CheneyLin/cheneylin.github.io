2. 需求背景与目标

商家活动期出现以下问题：

  • 限量商品（独特门票礼包/SPU）会被超级用户多次抢购导致普通用户无法购买；
  • 现有系统未提供可配置的人群包限购策略，无法做到个性化限购；
  • 规则管理零散（促销/商品模块无法覆盖限购冲突与精细化控制），导致支持成本高。
目标类型描述衡量指标目标值
业务稳定限购规则零误判限购相关拦截错误≤00
体验风险用户下单失败提示明确相关提示正确率≥99%99%
活动控制避免超卖/抛单有效库存一致性≥99.9%99.9%

6. 详细产品方案

6.1 商家后台模块 1：人群包策略管理

6.1.1 人群包策略管理（crowd-pack-module） crowd-pack-module

功能描述：商品中心新增人群包策略模块，用于管理和配置动态人群包。

模块名称

人群包策略

列表展示字段
  • 策略 ID
  • 人群包
  • 商品范围（SPU/SKU）
  • 加入条件
  • 上线状态
顶部操作按钮
  • 新建人群包策略
列表操作按钮
  • 编辑：编辑当前策略配置
  • 回溯：回溯 365 天内符合条件的用户
  • 上线/下线：快速上线/下线当前策略
  • 复制：复制当前策略
新增/编辑页面字段
  • 人群包名称：必填项，用于标识人群包
  • 绑定商品：SPU 限购/SKU 限购，默认选中 SPU 限购，为必填项
  • 批量 ID：多个商品 ID/多个 SKU ID，一行一个输入
  • 加入条件：支付完成、签收完成、订单完成单选
  • 状态：启用/禁用
初始化配置
  • 是否回溯：开启后回溯过去 365 天内购买记录
  • 退款剔除：由其他系统实现，本产品提供开关
数据流向
商家配置人群包策略 → 后台保存配置
→ 系统初始化人群（过去 365 天购买用户）
→ 动态更新（退款剔除等）

6.2 商家后台模块 2：限购策略管理

6.2.1 限购策略管理（limit-strategy-module） limit-strategy-module

功能描述：商品中心新增限购策略模块，用于配置和管理商品限购规则。

模块名称

限购策略

列表展示字段
  • 策略 ID
  • 名称
  • 商品范围（SPU/SKU）
  • 限购数量
  • 有效期
  • 状态
顶部操作按钮
  • 新建限购策略
列表操作按钮
  • 编辑：编辑当前策略配置
  • 上线/下线：快速上线/下线当前策略
  • 复制：复制当前策略
新增/编辑页面字段
  • 选择类别：SPU 限购/SKU 限购
  • 指定商品 ID：单个商品 ID
  • SKU 列表勾选：如果类别为 SKU 限购，展示 SKU 列表勾选界面，支持多个 SKU ID
  • 选择人群包：精确单选，默认不指定
  • 限购数量：正整数
  • 有效期：1 年/永久/自定义（到秒）
  • 冲突策略说明：自动说明"与当前策略同时命中时将取最小值"
  • 提示文案：可设置单条自定义文案
核心规则
  • 单策略仅支持单一人群包（多策略可覆盖不同人群）
  • 仅支持指定 SPU/SKU（多选）
  • 若多条策略同时命中，取最小限购值；任意一个策略不满足即拒单（严格执行）
  • 下单环节校验，提示来自策略配置字段
数据流向
商家配置限购策略 → 后台保存策略配置
→ 下单时查询匹配策略 → 计算限购值（取最小值）
→ 校验购买数量 → 返回结果（通过/拒绝）

6.3 APP 商品详情页面

6.3.1 APP 下单校验（app-checkout-module） app-checkout-module

功能描述：用户在客户端提交订单时，系统进行限购策略校验。

下单流程
  1. 校验购买数量：检查历史购买量（含已下单 + 待支付订单）是否超限，若超限则返回主动提示并拒绝
  2. 过滤策略：商品命中策略集、人群包匹配、生效期内
  3. 策略冲突：必须符合全部有效策略
校验条件
  • 当前用户与商品是否满足策略的 SPU/SKU 与人群包
  • 购买数量≤策略限购
  • 在生效期内
失败处理
  • 返回"策略提示文案"，阻断流程
  • 记录拦截日志（含策略 ID、人群包、商品、时间）
数据流向
用户提交订单 → 获取用户人群包信息
→ 查询匹配的限购策略 → 计算限购值（取最小值）
→ 检查历史购买量 → 校验购买总量
→ 返回结果（通过/拒绝并提示）
