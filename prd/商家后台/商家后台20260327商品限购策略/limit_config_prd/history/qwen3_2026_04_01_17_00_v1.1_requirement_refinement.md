### 三、APP 下单流程优化（5.3 章节）

#### 3.1 Markdown PRD 源文件
**文件**: `v1.1/prd_md/prd_v1.1.md`

**变更前**:
```markdown
### 5.3 APP 商品详情页面
- 下单流程：
  1. 读取购物车待下单商品、用户人群包规格。
  2. 过滤策略：商品命中策略集、人群包匹配、生效期内。
  3. 策略冲突：筛选符合的多策略，取限购值最小，作为用户可买上限。

```

**变更后**:
```markdown
### 5.3 APP 商品详情页面
- 下单流程：
  1. 校验购买数量（含已下单 + 待支付历史）是否超限；若超限则返回主动提示并拒绝。
  2. 过滤策略：商品命中策略集、人群包匹配、生效期内。
  3. 策略冲突：必须符合全部有效策略。
```

**优化说明**:
- ✅ **步骤 1 前置**: 将购买数量校验提前到第一步，快速失败

### 下单校验优化逻辑
```javascript
// 优化后的下单校验流程
async function checkoutValidation(orderData) {
  // 步骤 1: 快速校验购买数量（前置）
  const purchaseCount = await getHistoricalPurchaseCount(
    orderData.userId,
    orderData.itemId
  );
  
  if (purchaseCount >= config.maxLimit) {
    return {
      success: false,
      message: '您已达到购买上限',
      code: 'PURCHASE_LIMIT_EXCEEDED'
    };
  }
  
  // 步骤 2: 过滤匹配的策略
  const strategies = await filterStrategies({
    itemId: orderData.itemId,
    userTags: await getUserTags(orderData.userId),
    currentTime: new Date()
  });
  
  // 步骤 3: 校验是否符合全部有效策略
  const allStrategiesPassed = strategies.every(strategy => 
    validateStrategy(strategy, orderData)
  );
  
  if (!allStrategiesPassed) {
    return {
      success: false,
      message: '不符合限购策略要求',
      code: 'STRATEGY_VALIDATION_FAILED'
    };
  }
  
  return { success: true };
}
```
