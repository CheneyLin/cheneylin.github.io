## 变更对比分析

### 埋点事件精简
| 原事件 | 新事件 | 说明 |
|-------|-------|------|
| `event_strategy_view` | ❌ 移除 | 策略列表访问统计非核心指标 |
| `event_strategy_change` | ❌ 移除 | 配置操作已包含在提示事件中 |
| `event_order_limit_hit` | ❌ 移除 | 限购拦截场景统一归并 |
| `event_order_limit_pass` | ❌ 移除 | 下单校验通过无需单独埋点 |
| - | ✅ **新增** `limit_config_alert` | **统一的人群包策略/限购策略提示事件** |

---

### 精简原因分析

#### 优势
1. **聚焦核心指标**: 仅保留最关键的策略提示事件
2. **降低维护成本**: 减少 75% 的埋点数量（4→1）
3. **简化数据分析**: 单一事件便于统计和趋势分析
4. **统一事件命名**: `limit_config_alert` 更直观反映业务含义

#### 可能的影响
1. **历史数据对比**: v1.0 与 v1.1 的埋点数据无法直接对比
2. **细粒度分析缺失**: 无法区分策略列表访问、配置操作、下单校验等细分场景
3. **问题定位难度**: 出现问题时难以精确定位到具体环节

## 埋点事件详细说明

### limit_config_alert

#### 触发场景
1. **人群包策略提示**
   - 人群包策略启用/禁用
   - 人群包策略配置保存成功/失败
   - 人群包回溯操作完成

2. **限购策略提示**
   - 限购策略启用/禁用
   - 限购策略配置保存成功/失败
   - 限购策略冲突提示

3. **下单环节提示**
   - 用户下单时触发限购拦截
   - 策略失效提示
   - 人群包匹配失败提示

#### 事件参数建议
```javascript
// 示例数据结构
{
  event_name: 'limit_config_alert',
  event_time: '2026-04-01 16:45:00',
  user_id: 'user_12345',
  alert_type: 'strategy_conflict', // 提示类型
  strategy_id: 'S0001', // 策略 ID
  strategy_type: 'limit_strategy', // 策略类型：crowd_pack / limit_strategy
  action: 'create' | 'edit' | 'enable' | 'disable' | 'checkout', // 操作类型
  result: 'success' | 'failure', // 操作结果
  message: '与当前策略同时命中时将取最小值' // 提示文案
}
```

## 技术实现建议

### 前端埋点代码示例
```javascript
// 统一的策略提示埋点函数
function trackLimitConfigAlert(params) {
  const eventData = {
    event_name: 'limit_config_alert',
    event_time: new Date().toISOString(),
    user_id: getCurrentUserId(),
    ...params
  };
  
  // 发送到埋点服务器
  sendToAnalytics(eventData);
  
  // 本地日志记录
  console.log('[Limit Config Alert]', eventData);
}

// 使用示例
// 1. 人群包策略保存成功
trackLimitConfigAlert({
  alert_type: 'strategy_saved',
  strategy_id: 'C001',
  strategy_type: 'crowd_pack',
  action: 'create',
  result: 'success'
});

// 2. 限购策略冲突提示
trackLimitConfigAlert({
  alert_type: 'strategy_conflict',
  strategy_id: 'S0001',
  strategy_type: 'limit_strategy',
  action: 'save',
  result: 'warning',
  message: '与当前策略同时命中时将取最小值'
});

// 3. 下单限购拦截
trackLimitConfigAlert({
  alert_type: 'checkout_blocked',
  strategy_id: 'S0001',
  strategy_type: 'limit_strategy',
  action: 'checkout',
  result: 'failure',
  message: '您已达到购买上限'
});
```
