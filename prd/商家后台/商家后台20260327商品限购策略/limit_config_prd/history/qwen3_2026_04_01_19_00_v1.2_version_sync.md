## 版本管理规范

### 版本号命名规则
- **格式**: `v{主版本}.{次版本}`
- **示例**: v1.0, v1.1, v1.2
- **语义**: 
  - 主版本号：重大架构变更
  - 次版本号：功能迭代、需求优化

### 版本迭代记录要求
1. **必填字段**: 版本号、日期、变更项、负责人
2. **变更日期**: 精确到日（YYYY-MM-DD）
3. **变更项**: 简明扼要概括核心变更（≤20 字）
4. **负责人**: 实际贡献者姓名或 AI 助手名称

### 全链路一致性检查
每次版本升级需同步更新以下位置：
1. ✅ Markdown PRD 源文件标题
2. ✅ HTML PRD 文档标题和元数据
3. ✅ 后台原型页面标题
4. ✅ APP 原型页面标题（如有）
5. ✅ 主控文档版本选择器
6. ✅ 主控文档 iframe 路径
7. ✅ 版本徽章显示
