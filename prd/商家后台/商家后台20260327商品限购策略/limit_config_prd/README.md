# 商品限购策略PRD项目

## 项目概述

本项目是商家后台商品限购策略的完整PRD（产品需求文档）包，包含需求分析、详细设计、交互原型和业务流程图。

## 快速开始

### 1. 打开主控文档
直接打开 `index.html` 文件启动PRD查看器。

### 2. 功能说明
- **版本切换**：选择不同的PRD版本
- **产品形态切换**：在后台管理（Admin）和移动APP之间切换
- **PRD联动**：点击PRD文档中的按钮可高亮对应的原型功能

## 项目结构

```
商品限购策略_PRD/
├── index.html                    # 主控文档（入口文件）
├── delivery_checklist_v1.0.md    # 交付清单
├── README.md                     # 项目说明（本文件）
├── history/                      # 历史版本存档
├── resource/                     # 公共资源文件
└── v1.0/                        # v1.0版本
    ├── prd_md/                  # Markdown格式PRD
    ├── prd_html/                # HTML格式PRD（推荐查看）
    ├── prototype_html/          # 交互原型
    └── assets/                  # 资源文件
```

## 核心功能

### 限购策略规则
- **单策略单人群包**：每个策略仅支持一个用户人群包
- **多商品支持**：支持SPU/SKU多选配置
- **冲突处理**：多策略命中时取最小限购值
- **灵活有效期**：支持永久、1年、自定义时间范围

### 用户体验
- **明确提示**：限购失败时显示配置的提示文案
- **实时校验**：下单时进行策略校验
- **数据埋点**：完整的用户行为追踪

## 技术栈

- **文档格式**：HTML + Markdown
- **交互原型**：Vue.js + Element UI（后台）、Tailwind CSS（APP）
- **流程图**：Mermaid
- **联动技术**：postMessage API

## 使用说明

### 查看PRD文档
1. 打开 `index.html`
2. 左侧显示PRD文档，右侧显示对应原型
3. 使用版本选择器切换不同版本
4. 使用产品形态按钮切换后台/APP视图

### PRD与原型联动
1. 在PRD文档中找到蓝色按钮
2. 点击按钮可高亮对应的原型功能
3. 原型会自动切换到对应页面并高亮显示

### 原型交互
- **后台原型**：支持表单填写、列表查看、页面导航
- **APP原型**：模拟移动端购买流程和限购提示

## 开发说明

### 本地部署
无需特殊服务器环境，直接用浏览器打开HTML文件即可。

### 文件修改
- PRD文档：编辑 `prd_html/prd_v1.0.html` 或 `prd_md/prd_v1.0.md`
- 原型文件：编辑 `prototype_html/` 目录下的对应文件
- 主控文档：编辑 `index.html`

### 版本管理
- 新版本创建新的版本目录（如 `v2.0/`）
- 在 `index.html` 的版本选择器中添加新选项
- 历史版本移至 `history/` 目录

## 常见问题

### Q: 原型显示不正常？
A: 确保网络连接正常，外部CSS/JS库能正常加载。

### Q: 联动功能不工作？
A: 检查浏览器是否支持postMessage API，尝试刷新页面。

### Q: 流程图不显示？
A: 确保网络连接正常，Mermaid库能正常加载。
