## 2. 需求背景与目标

### 2.1 现状问题与痛点

当前芒芒赏业务存在以下痛点：

1. **供货价编辑权限受限**：芒芒赏实物商品的供货价写死为0，不可编辑，无法支持不同SKU设置不同成本价的业务需求
2. **财务成本计算不精确**：当前模式下，用户购买芒芒赏活动商品时同时产生一个虚拟订单和一个实物订单，收款和成本均记录在虚拟订单上，无法体现实物商品不同SKU的实际成本差异
3. **财务模式单一**：缺少"收款记录在虚拟订单，成本记录在实物订单"的财务模式，影响订单结算和财务审核的精确性

### 2.2 需求提出的必要性

通过本次优化，可以：
- 支持实物商品不同SKU设置不同成本价，实现精细化成本管理
- 提供更灵活的财务模式选择，确保财务数据严谨
- 为订单结算和财务审核提供精确的数据基础

### 2.3 业务目标

| 目标类型 | 描述 | 衡量指标 | 目标值 |
| --- | --- | --- | --- |
| 财务精确性 | 确保芒芒赏活动虚拟订单和实物订单不重复计算供货价 | 供货价重复计算错误数 | 0 |
| 成本管理 | 支持实物商品SKU级别的成本价配置 | SKU供货价配置覆盖率 | 100% |
| 数据安全 | 通过校验机制防止供货价配置错误 | 活动保存校验拦截率 | 100% |

### 2.4 用户目标

| 目标类型 | 描述 | 衡量指标 |
| --- | --- | --- |
| 运营效率 | 运营人员能够快速为实物商品配置SKU级别供货价 | 商品编辑操作完成时间 |
| 财务准确 | 财务人员能够获取准确的成本数据 | 财务数据校验通过率 |

## 3. 用户与使用场景

### 3.1 核心用户

- **自营/一件代发店铺运营人员**：负责芒芒赏活动的商品配置和活动管理（店铺类型为"自营"或"一件代发"）
- **财务审核人员**：负责订单结算和财务数据审核

### 3.2 典型场景

1. 运营人员在商品平台编辑芒芒赏实物商品，为不同SKU配置供货价
2. 运营人员在营销模块新建芒芒赏活动，选择虚拟商品和实物商品
3. 系统在保存活动时校验两个商品的供货价配置是否符合互斥规则
4. 财务人员在订单结算时查看准确的成本数据

### 3.3 核心用户旅程

| 阶段 | 用户触点 | 用户行为 | 痛点/情绪 | 产品机会 |
| --- | --- | --- | --- | --- |
| 商品配置 | 商品平台-商品编辑页 | 编辑芒芒赏实物商品，为SKU配置供货价 | 困惑：供货价字段不可编辑 | 放开供货价编辑权限，支持SKU级别配置 |
| 活动创建 | 营销-芒芒赏-新建活动 | 选择虚拟商品和实物商品，填写活动信息 | 担忧：不确定供货价配置是否正确 | 保存时自动校验供货价互斥规则 |
| 校验反馈 | 活动保存弹窗 | 系统校验供货价配置 | 焦虑：配置错误导致活动无法保存 | 提供明确的错误提示信息 |
| 财务结算 | 订单结算/财务审核 | 查看虚拟订单和实物订单的成本数据 | 不满：成本数据不精确 | 支持成本记录在实物订单上的新模式 |

## 5. 详细方案

### 5.1 商品平台 - SKU级别供货价编辑（admin_product_edit_supply_price）

**功能描述**：在商品编辑页面中，支持为芒芒赏实物商品和虚拟商品的SKU配置供货价。供货价编辑统一在SKU级别进行，实物商品和虚拟商品的区别在于SKU数量和是否允许添加。

**详细逻辑**：
- 权限控制：仅当店铺类型为"自营"或"一件代发"且商品类型为"芒芒赏实物商品"或"芒芒赏虚拟商品"时，SKU供货价字段可编辑
- 其他店铺或其他商品类型，SKU供货价字段保持原有逻辑不变
- 供货价输入限制：仅支持正数
- **芒芒赏实物商品**：
  - 支持多个SKU，每个SKU可单独配置不同的供货价
  - 支持添加新SKU，新添加的SKU也需要配置供货价
  - SKU供货价允许为空（不设置），表示该SKU不单独配置成本价
  - 当实物商品选择作为芒芒赏活动的成本载体时，所有SKU都必须配置大于0的供货价
- **芒芒赏虚拟商品**：
  - 仅有1个SKU，不支持添加新SKU
  - 该SKU的供货价输入框可编辑
  - 虚拟商品供货价即为该唯一SKU的供货价

**交互设计**：
- SKU列表使用表格形式展示，包含SKU名称、规格、供货价等列
- 供货价列使用输入框，支持直接编辑
- 芒芒赏实物商品：表格底部显示"添加SKU"按钮，删除SKU时需二次确认
- 芒芒赏虚拟商品：不显示"添加SKU"按钮，不显示删除操作
- 输入非正数时，前端实时校验并提示

**数据流向**：
```
运营进入商品编辑页 → 系统判断店铺ID和商品类型 → 加载SKU列表
→ 展示SKU供货价编辑区域 → 运营编辑SKU供货价 → 前端校验 → 保存至SKU数据表
```

### 5.2 营销-芒芒赏-新建活动供货价校验（admin_activity_create_validate）

**功能描述**：在新建芒芒赏活动保存时，对关联的虚拟商品和实物商品的供货价配置进行互斥校验，确保有且仅有一个商品正确配置供货价。

**详细逻辑**：
- **互斥规则**：虚拟商品和实物商品必须且只能有一个商品设置供货价（供货价 > 0）
- **完整性规则**：若实物商品配置供货价，则该实物商品的所有SKU都必须配置大于0的供货价
- **校验时机**：仅在新建芒芒赏活动点击保存时触发
- **校验逻辑流程**：
  1. 检查虚拟商品供货价是否 > 0
  2. 检查实物商品是否有任一SKU供货价 > 0
  3. 判断两者是否同时存在供货价 > 0（违反互斥规则）
  4. 判断两者是否都不存在供货价 > 0（违反必须且只能有一个的规则）
  5. 若实物商品有供货价，检查是否所有SKU供货价均 > 0（违反完整性规则）

**交互设计**：
- 校验通过：正常保存活动，展示成功提示
- 校验失败：阻断保存，弹出标准错误提示弹窗

**数据流向**：
```
运营点击保存活动 → 系统获取虚拟商品和实物商品供货价数据
→ 执行互斥校验和完整性校验
→ 校验通过：保存活动数据
→ 校验失败：返回错误信息，弹窗提示
```

### 5.3 营销-芒芒赏-新建活动校验失败提示（admin_activity_create_error_tip）

**功能描述**：当新建芒芒赏活动保存时供货价校验失败，展示标准弹窗提示用户修正配置。

**详细逻辑**：
- 使用标准弹窗组件展示错误信息
- 错误提示文案："供货价配置错误，请检查，确保有且仅有一个商品正确配置供货价。"
- 弹窗包含"确定"按钮，点击后关闭弹窗，用户可返回修改配置
- 弹窗不可通过点击外部区域关闭

**交互设计**：
- 标准错误弹窗，居中展示
- 弹窗标题："提示"
- 弹窗内容：错误提示文案
- 弹窗按钮："确定"（主按钮）

**数据流向**：
```
校验失败 → 触发错误弹窗 → 展示错误信息
→ 用户点击确定 → 关闭弹窗 → 用户修改配置
```

## 7. 异常与边界处理

- **非目标店铺访问**：店铺类型非"自营"或"一件代发"的用户编辑芒芒赏实物商品时，SKU供货价字段保持不可编辑状态，不展示任何额外提示
- **供货价输入非法值**：输入非正数时，前端实时校验，输入框标红并提示"请输入正数"
- **SKU供货价部分配置**：实物商品部分SKU配置了供货价、部分未配置时，若该商品被选为芒芒赏活动的成本载体，保存活动时校验失败并提示
- **网络异常**：保存商品或活动时网络中断，保留表单数据，提示"保存失败，请重试"
- **并发编辑**：同一商品被多人同时编辑时，以后保存者为准，前端提示"商品信息已被他人修改，请刷新后重试"
- **活动审核/上线状态**：活动进入审核或上线状态后，商品设置不可编辑（已有逻辑，本次不涉及修改）
