2. 需求背景与目标

2.1 现状问题与痛点

当前愚公互动活动平台存在以下痛点：

  1. 活动互动形式单一：现有活动页面缺乏实时互动展示组件，用户参与感和沉浸感不足
  2. 话题内容利用率低：社区话题帖子仅在社区内展示，无法有效引流到活动页
  3. 活动氛围营造弱：明星应援等活动缺少实时滚动的弹幕氛围，用户无法快速了解活动内容
  4. 创作者工具不完善：活动创作者在搭建页面时，缺少弹幕墙这一常见互动组件

2.2 需求提出的必要性

通过引入"弹幕墙组件"功能，可以：

  • 丰富活动页面的互动展示形式，提升用户参与度
  • 将社区话题内容有效引流到活动页，形成内容消费闭环
  • 为明星应援等活动营造实时互动氛围
  • 完善活动创作工具链，降低活动搭建成本

3. 用户与使用场景

3.1 核心用户

  • 活动创作者：运营人员、品牌方、活动主办方（使用后台H5编辑器搭建活动页）
  • 活动参与者：明星粉丝、活动用户（浏览活动页面，查看弹幕互动）

3.2 典型场景

  1. 活动创作者在后台H5编辑器中拖拽添加弹幕墙组件
  2. 创作者通过关键字搜索话题，多选添加到弹幕墙配置
  3. 创作者配置弹幕样式、排序方式
  4. 保存并预览活动页，查看弹幕实时滚动效果
  5. 活动发布后，用户访问活动页看到弹幕高密度自动滚动
  6. 用户点击弹幕可跳转查看完整帖子内容（如有）

5. 详细方案

5.1 H5编辑器端功能

5.1.1 弹幕墙组件拖拽添加

功能描述：活动创作者从组件面板拖拽弹幕墙组件到H5画布中。

详细逻辑
  • 在H5编辑器的组件面板中新增"弹幕墙"组件图标
    • 组件图标使用弹幕相关视觉元素（如滚动文字图标）
    • 支持拖拽到画布任意位置
    • 拖拽后默认展示弹幕墙占位框（宽100%，高度可调整）
    • 占位框内显示"弹幕墙组件"文字和配置提示
  • 配置面板
    • 话题搜索输入框
      • 输入关键字后（≥2个字符），自动触发搜索建议
      • 已选择的话题以标签（Tag）形式展示在输入框下
      • 支持删除已选择的话题
      • 支持添加多个话题
    • 行数输入框 - 设置弹幕显示行数
    • 样式下拉框
      • 样式1：仅标题 - 只展示帖子标题文字
      • 样式2：标题+内容摘要 - 展示标题和最多50字内容摘要
      • 样式3：用户头像+标题 - 左侧展示用户头像，右侧展示标题
      • 样式4：用户头像+标题+内容摘要 - 完整展示头像、标题、内容摘要
    • 排序下拉框
      • 按点赞数排序 - 点赞数高的帖子优先展示
      • 按最新发布时间排序 - 最新发布的帖子优先展示
交互设计
  • 拖拽时显示组件预览缩略图
  • 放置到画布后自动选中并弹出右侧配置面板
  • 组件支持调整大小和位置（与其他H5组件一致）
数据流向
用户拖拽组件 → 添加到画布 → 生成组件实例ID → 展示配置面板

5.2 C端活动页面功能

5.2.1 弹幕自动滚动展示

功能描述：活动页面高密度自动滚动展示弹幕。

详细逻辑
  • 弹幕从右向左自动滚动播放
  • 高密度展示：同时展示多行弹幕（默认5-8行）
  • 弹幕滚动速度适中（约8-12秒完成一次完整滚动）
  • 弹幕之间保持适当间距，避免重叠
  • 滚动流畅无卡顿，支持CSS动画优化性能
  • 弹幕展示内容根据后台配置的样式组合决定
交互设计
  • 弹幕行使用不同垂直位置，营造层次感
  • 新弹幕从右侧平滑进入，离开左侧后销毁
  • 弹幕支持鼠标悬停暂停（可选，根据活动需求配置）
  • 高密度展示营造热烈氛围
数据流向
页面加载 → 获取弹幕配置 → 请求话题帖子数据 → 渲染弹幕行 → 启动滚动动画

7. 异常与边界处理

  • 空状态处理：
    • 话题无帖子：显示"暂无话题内容，快来参与讨论吧！"
    • 未配置话题ID或配置错误：也显示以上提示
  • 弹幕数量不足：帖子数量少于弹幕行时，循环展示已有帖子
  • 性能边界：
    • 单组件最多同时展示100条弹幕（超过则排队展示）
    • 弹幕刷新间隔最低30秒，避免频繁请求
    • 组件最大高度限制为页面高度80%，超出部分可滚动
