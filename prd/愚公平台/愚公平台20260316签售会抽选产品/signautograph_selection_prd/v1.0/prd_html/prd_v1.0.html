一、需求背景

1.1 现状问题

明星线下专辑签售会需要对购买专辑的用户进行抽选，并生成名单公布。目前该流程缺乏系统化产品支持，运营人员需要手工处理数据导出、抽选和名单公布，效率较低且容易出错。

1.2 痛点分析

  1. 数据处理繁琐：运营人员需要手动导出 CSV 数据，手工筛选和处理
  2. 抽选过程不透明：缺乏可视化的抽选展示，难以保证公平性
  3. 结果公布效率低：名单整理和发布耗时长，易出错
  4. 无法追溯审计：缺少操作记录，难以追溯抽选过程

1.3 必要性说明

通过产品化实现抽选流程的标准化、自动化和可视化，提高运营效率，确保抽选过程的公平性和透明度。

二、需求目标

目标类型 描述 衡量指标 目标值
效率提升 缩短抽选活动准备时间 从数据准备到结果公布的总耗时 ≤30 分钟
准确性 确保抽选结果准确无误 数据导入准确率、抽选结果准确率 ≥99.9%
用户体验 简化运营人员操作流程 操作步骤数、学习成本 ≤5 步核心操作
透明度 提升抽选过程可视化程度 支持全屏展示、过程可追溯 100% 支持
灵活性 支持多次调整重选 支持重新导入用户包、调整规则 无限次重选

三、用户与使用场景

3.1 目标用户画像

核心用户：运营人员
• 基本特征：负责明星签售会活动的组织和执行
• 技能水平：熟悉基本的后台操作和 Excel 使用
• 使用频率：按活动周期使用（不定期，每次活动使用 1 次）
• 核心诉求：快速完成抽选、确保结果准确、流程可追溯

3.2 典型使用场景

场景 1：新活动抽选配置

  • 触发条件：收到新的签售会活动规则和通知
  • 用户目标：完成活动场次配置，导入用户数据，设置抽选规则
  • 使用环境：办公室，PC 端后台系统

场景 2：执行抽选操作

  • 触发条件：到达预定抽选时间，所有准备工作完成
  • 用户目标：执行抽选，查看抽选结果，必要时调整规则重选
  • 使用环境：会议室或办公室，可能需要投屏展示

场景 3：结果导出与公布

  • 触发条件：抽选完成，确认最终结果
  • 用户目标：导出中签名单，设置为对外公布状态
  • 使用环境：办公室，完成后续发布流程

五、详细方案

5.1 愚公平台 - 后台管理端

5.1.1 运营工具箱 → 签售会抽选

5.1.1.1 创建活动场次 👁️ 查看原型

功能描述：从运营工具箱入口进入，创建新的签售会抽选活动

前置条件：用户已登录后台管理系统，具有运营工具箱访问权限

业务规则：

  1. 活动名称必须唯一，不可重复
  2. 活动时间必须在当前时间之后
  3. 抽选名额必须为正整数（范围 1-99999）
  4. 活动状态默认为"筹备中"
  5. 创建成功后自动进入活动编辑页面

输入字段：

  • 活动名称（必填，文本，最多 50 字符）
  • 活动时间（必填，日期时间选择器）
  • 活动地点（必填，文本，最多 100 字符）
  • 抽选名额（必填，数字，范围 1-99999）
  • 活动备注（选填，文本，最多 500 字符）

交互流程：

  1. 点击运营工具箱中的"签售会抽选"入口
  2. 点击"创建活动"按钮
  3. 弹出活动创建表单
  4. 填写必填信息
  5. 系统校验数据合法性
  6. 提交成功，跳转至活动详情页（列表管理 - 编辑）

异常处理：

  • 活动名称重复：提示"该活动名称已存在，请使用其他名称"
  • 时间在过去：提示"活动时间必须晚于当前时间"
  • 名额超出范围：提示"抽选名额必须在 1-99999 之间"

5.1.1.2 活动列表管理 👁️ 查看原型

功能描述：展示所有抽选活动列表，支持筛选和快捷操作

页面元素：

  • 搜索区：搜索框、状态筛选、时间筛选
  • 列表字段：活动名称、时间、地点、名额、状态、创建时间、操作

操作按钮：

  • 查看详情（所有状态可用）
  • 编辑活动（仅筹备中状态可用）
  • 删除活动（仅筹备中状态可用，需二次确认）
  • 进入抽选（仅已导入用户包的活动可用）

5.1.1.3 编辑活动信息 👁️ 查看原型

功能描述：修改已创建活动的详细信息

前置条件：活动处于"筹备中"状态

业务规则：

  1. 仅筹备中状态的活动可以编辑
  2. 修改后自动保存（或手动点击保存）
  3. 活动名称修改后仍需保持唯一性
  4. 已开始的活动不允许修改

可编辑字段：活动名称、活动时间、活动地点、抽选名额、活动备注

5.1.1.4 用户包列表管理 👁️ 查看原型

功能描述：二级页面，展示当前活动的所有用户包列表

页面元素：页面标题、返回列表按钮、新增按钮、列表字段（名称、来源、数量、时间、状态、操作）

业务规则：

  1. 一个活动可以有多个用户包
  2. 同一时间只能有一个用户包处于"已启用"状态
  3. 启用新用户包时，自动停用旧用户包
  4. 已有抽选结果的用户包不允许删除

5.1.1.5 新增用户包 👁️ 查看原型

功能描述：上传 Excel/CSV 格式的用户数据文件

数据格式要求：

序号，用户 UUID, 用户昵称，手机号码，身份证号码，身份证姓名，实收，件数，时间，邮箱信息，TO 签内容
1,db98973...,Peach,13706589999,330402200001011200，陶缘，1999,1,2026-01-02 12:00:00,-,-

必填字段：用户 UUID、手机号码、身份证姓名

文件大小限制：≤50MB

数据校验：文件格式、必填字段、数据格式、文件过大、重复数据

5.1.1.6 配置抽选规则 👁️ 查看原型

功能描述：配置抽选规则、逻辑、轮次，支持多轮抽选

前置条件：已导入用户包且用户包处于已启用状态

业务规则：

  1. 默认 1 轮抽选
  2. 支持配置多轮抽选（如第一轮 50 人，第二轮 30 人）
  3. 每轮可独立配置：名称、人数、使用的用户包、白名单
  4. 可设置是否允许重复中签（默认不允许）

抽选逻辑：

  • A1-A3：按用户随机（各抽各的/剔除上轮/末轮剔除）
  • B1-B3：按金额取前（各抽各的/剔除上轮/末轮剔除）
  • C1-C3：按件数权重随机（各抽各的/剔除上轮/末轮剔除）

5.1.1.7 抽选列表管理 👁️ 查看原型

功能描述：二级页面，展示所有抽选记录，支持新增抽选、导出、回放、设为公开

页面元素：页面标题、返回列表按钮、新增抽选按钮、列表字段（轮次、时间、人数、状态、操作）

操作按钮：

  • 新增抽选：点击进入全屏抽选模式
  • 导出：导出该轮抽选结果为 Excel 文件
  • 回放：重新播放该轮抽选的动画过程
  • 设为公开：将该轮结果设置为对外公布状态（需二次确认）

5.1.1.8 全屏抽选展示 👁️ 查看原型

功能描述：进入沉浸式全屏抽选界面，展示抽选过程

界面布局：

  • 顶部区域：活动名称、抽选轮次、规则摘要、退出按钮
  • 中央展示区：用户信息滚动展示、炫酷动画效果
  • 底部控制区：开始/暂停/停止按钮组、进度显示、结果缩略图

视觉效果：深色背景（#1a1a2e）、渐变紫色主题、卡片式展示、平滑动画、庆祝特效

业务规则：

  1. 点击"开始"时，后端立即计算完整抽选结果
  2. 前端动画仅为展示效果，不影响结果
  3. 确保结果已定，过程可控
  4. 支持中途暂停（视觉暂停，非结果暂停）
  5. 点击"停止"后结果不可更改

5.1.1.9 通知列表管理 👁️ 查看原型

功能描述：配置通知任务，向指定人群发送短信、PUSH、站内信等

前置条件：已有中签用户

配置项：

  • 基础配置：任务名称、通知类型、目标人群
  • 文案配置：短信文案（500 字符）、PUSH 文案、站内信文案
  • 发送配置：发送时间、发送频率、签名配置

示例文案：

【小芒 APP】亲爱的{用户昵称}，恭喜您在{活动名称}中获得中签资格！
请于{截止时间}前前往{活动地点}参加签售会。
