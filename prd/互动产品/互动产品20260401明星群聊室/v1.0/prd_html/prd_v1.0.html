一、需求背景

1.1 现状问题

问题类型问题描述影响范围紧急程度
粉丝互动不足现有直播弹幕互动形式单一，粉丝缺乏与艺人深度交流的渠道平台自有艺人及粉丝群体高
艺人变现受限艺人缺少有效的粉丝运营工具和商业化途径平台签约艺人高
用户粘性低缺乏持续吸引用户的独家内容和互动机制平台整体活跃度中
竞品压力韩国 Bubble 等产品已验证艺人私聊模式，国内市场需求未被满足市场份额流失风险中

1.2 需求提出的必要性

  1. 市场机会：粉丝经济市场规模持续增长，用户付费意愿强烈，14-40 岁核心用户群体具有较强消费能力
  2. 差异化竞争：相比直播产品的弹幕互动，提供私密、深度、长期的互动体验
  3. 平台优势：小芒 App 已有自有艺人资源，可快速冷启动；现有会员体系可复用
  4. 商业价值：订阅制模式带来稳定现金流，提升艺人商业价值和平台收入

五、详细产品方案

注：完整详细方案包含 29 个功能的详细说明，此处展示核心功能框架。详细内容请参考 Markdown 源文档。

5.1 移动端（小芒 App）- 核心功能

5.1.1 VIP 产品购买与订阅管理 vip_purchase_01

功能描述：用户购买指定艺人的 VIP 产品，获得与该艺人互动的权益。

业务逻辑
  • 用户在艺人详情页点击"订阅 VIP"
  • 选择套餐类型（单月 28 元 / 连续订阅 20 元/月）
  • 确认订单信息并调用支付接口
  • 支付成功后开通 VIP 权限，自动加入群聊室
业务规则
  • 连续订阅默认开启自动续费，用户可随时取消
  • VIP 到期前 3 天、1 天分别发送续费提醒
  • 购买后 7 天内未使用可申请退款

5.1.2 专属群聊室 group_chat_01

功能描述：每个艺人一个官方大群，购买 VIP 后自动获得入群资格。

权限矩阵
操作艺人（群主）管理员普通会员
发送文字/图片✅✅✅
撤回他人消息✅✅❌
禁言/踢出成员✅✅❌

5.1.3 一对一私信聊天 private_chat_01

功能描述：用户可向已订阅的艺人发送私信，艺人可在专用界面查看并回复。

核心特性
  • 支持文字、表情、图片、语音消息
  • 智能分类：高优先级消息优先展示
  • AI 助理辅助回复（标注"AI 提供"）

5.2 运营后台（Web）- 核心功能

5.2.1 艺人入驻审核 admin_artist_audit_01

功能描述：运营人员审核艺人入驻申请，签署电子合约。

审核流程
艺人提交申请 → 运营初审 → 运营复审 → 签署合约 → 开通权限

5.2.2 订单查询与导出 admin_order_query_01

功能描述：按条件查询订单详情，支持导出 Excel/CSV 格式。

查询条件
  • 用户 ID/手机号、艺人 ID/姓名
  • 订单时间范围、订单状态
  • 支付方式（微信/支付宝/余额）

六、异常与边界处理

场景处理方式用户提示
支付过程中断支付回调对账，自动重试或退款"支付结果确认中，请稍后查看订单状态"
发送消息失败本地保存草稿，网络恢复后提示重发"网络连接失败，消息已保存，点击重试"
VIP 到期保留历史消息查看权限，禁止发送新消息"VIP 已到期，续费后可继续与艺人互动>>"
无权限操作隐藏对应菜单或按钮（不展示明确提示）
