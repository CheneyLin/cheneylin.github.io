## 变更摘要（对齐 qq-prd 工作流）

本次变更聚焦 `v1.0` 的 **5.1.1 回填单（feature_01）**：以 `v1.0/prd_md/prd_v1.0.md` 为准，同步更新 PRD HTML 与 H5 原型，并修复/对齐主控联动入口。

## 变更内容

- **PRD（HTML）同步**
  - 更新 `v1.0/prd_html/prd_v1.0.html` 中 **5.1.1（feature_01）** 的呈现结构，使其与 Markdown 一致：
    - 明确“功能描述 / 详细逻辑 / 交互设计 / 数据流向”四块内容
    - 保留并强化“表单一次性完整展示（商品选择 → 收货地址 → 小票信息）”的核心约束

- **原型（H5）同步**
  - 更新 `v1.0/prototype_html/h5_v1.0.html`，将原本分散的 `product/address/receipt` 三页，合并为 **单页回填单 `#combined`**：
    - `feature_01`：回填单单页整体模块（商品选择 + 收货地址 + 小票信息）
    - `feature_02`：提交流程模块作为同页锚点，便于联动定位与高亮
  - 保留 `#success`（`feature_03`）与 `#feedback`（`feature_04`）用于提交结果展示与失败反馈。
  - 同步修复页面跳转按钮指向（成功页“继续回填”、反馈页“重试/返回并修改”均回到 `#combined`）。
  - 同步更新 Focus/Hash 逻辑：
    - `validPages` 调整为 `combined/success/feedback`
    - `feature_01/feature_02` 的 Focus 统一映射到 `combined`（并滚动定位对应模块）

- **主控（index.html）同步**
  - 更新根目录 `index.html`：右侧默认打开 H5 原型时直接进入 `#combined`，更贴合 5.1.1 的“单页回填单”定义。

## 联动验证点（建议）

- 在 `index.html` 左侧 PRD 点击：
  - `feature_01`：右侧应高亮 `#combined` 中回填单模块
  - `feature_02`：右侧应定位到同页的“提交流程”模块并高亮
  - `feature_03/feature_04`：分别定位到 `success/feedback` 并高亮对应模块
