抽盒机装盒算法规范

抽盒机装盒算法规范

一、装盒策略（6步完整流程）

步骤1: 初始化盒子结构

  • 输入参数：
    • 活动箱数（大箱数量）
    • 每箱盒数（每个大箱内的小盒数量）
    • 单盒商品数（每个小盒内的商品数量）
  • 计算：
    • 总盒数 = 活动箱数 × 每箱盒数
    • 总商品数 = 活动箱数 × 每箱盒数 × 单盒商品数
  • 输出：创建所有空盒子结构

步骤2: SKU分配装盒

  • 策略：将库存最多的SKU优先放入商品数量最少的单盒
  • 算法：
    1. 按库存数量降序排序所有SKU
    2. 找到当前商品数最少且未满的小盒
    3. 从库存最多的SKU中取出一个放入该小盒
    4. 重复直到所有库存分配完毕
  • 目标：均匀分布库存，避免某些盒子过早填满

步骤3: 隐藏款调剂

  • 策略：确保每个大箱至少有一个隐藏款，采用动态平衡原则
  • 算法：
    1. 统计每个大箱的隐藏款数量
    2. 找出拥有最多隐藏款的大箱（MaxBox）和拥有最少隐藏款的大箱（MinBox）
    3. 调剂条件：仅当 MaxBox 隐藏款数量 ≥ 2 且 MinBox 隐藏款数量 = 0 时执行
    4. 执行动作：将 MaxBox 中的一个隐藏款盒子与 MinBox 中的一个普通盒子交换
    5. 迭代执行：重复上述过程，直到无法找到满足条件的调剂对象
  • 目标：保证每个大箱都有隐藏款，提升用户体验

步骤4: 随机打乱大箱内盒子顺序

  • 策略：将每个大箱内的小盒顺序随机打乱
  • 算法：对每个大箱内的小盒使用 Fisher-Yates 洗牌算法
  • 目标：增加不确定性，避免隐藏款位置可预测

步骤5: 随机打乱单盒内商品顺序

  • 策略：将每个小盒内的商品SKU顺序随机打乱
  • 算法：对每个小盒内的商品使用 Fisher-Yates 洗牌算法
  • 目标：在微观层面增加随机性

步骤6: 随机打乱大箱顺序

  • 策略：将所有大箱的整体顺序随机打乱
  • 算法：
    1. 收集所有大箱的索引
    2. 使用 Fisher-Yates 洗牌算法打乱索引顺序
    3. 根据新顺序重新分配所有小盒的 boxIndex
  • 目标：在宏观层面增加随机性，完全打乱大箱排列

二、验证规则（4项必检）

装盒完成后，必须通过以下四项验证以确保结果正确：

验证1: 隐藏款分布

  • 检查项：每个大箱至少包含一个隐藏款
  • 验证方法：遍历所有大箱，检查是否每个大箱内至少有一个小盒包含隐藏款SKU
  • 失败示例：大箱3没有任何隐藏款
  • 重要性：⭐⭐⭐⭐⭐ 核心业务规则

验证2: 单盒唯一性

  • 检查项：同一个单盒内不能包含重复的SKU
  • 验证方法：检查每个小盒内的商品列表，确保没有重复的SKU ID
  • 失败示例：大箱1-盒2中有2个SKU_A
  • 重要性：⭐⭐⭐⭐⭐ 防止同一盒内出现相同商品

验证3: 库存全分配

  • 检查项：所有SKU的库存必须全部分配完毕，无剩余
  • 验证方法：
    • 计算总库存 = 所有SKU的stock字段之和
    • 计算已分配 = 所有小盒内商品数量之和
    • 验证：总库存 === 已分配
  • 失败示例：库存150个，已分配148个，差异2个
  • 重要性：⭐⭐⭐⭐⭐ 确保无库存浪费

验证4: 数量一致性

  • 检查项：每个单盒内的商品数量必须严格等于设定的单盒商品数
  • 验证方法：遍历所有小盒，检查 box.products.length === productsPerItem
  • 失败示例：大箱2-盒3有8个商品（应为10个）
  • 重要性：⭐⭐⭐⭐⭐ 保证产品规格一致

三、可视化展示规范

网格布局

  • 大箱展示：每个大箱独立显示，带边框和标题
  • 小盒网格：使用 CSS Grid 布局，响应式列宽（minmax(100px, 1fr)）
  • 紧凑设计：间距2px，最大化信息密度

SKU颜色编码

  • 隐藏款：红色 (#ff5b4f)，不透明度 1.0，带边框和阴影
  • 普通款：不同SKU使用不同颜色，不透明度 0.5
    • SKU_A: #0072f5 (蓝)
    • SKU_B: #10b981 (绿)
    • SKU_C: #f59e0b (琥珀)
    • SKU_D: #ef4444 (红)
    • SKU_E: #8b5cf6 (紫)
    • SKU_F: #ec4899 (粉)
    • SKU_G: #06b6d4 (青)
    • SKU_H: #84cc16 (lime)
    • SKU_I: #f97316 (橙)
    • SKU_J: #6366f1 (靛蓝)

状态标识

  • 隐藏款指示器：★ 符号（红色）
  • 含隐藏款的小盒：红色边框 + 浅红背景
  • 空盒：显示"空"文字（灰色）

分步展示

每一步操作后都展示当前状态的可视化结果：

  1. 初始状态（空盒子）
  2. SKU分配完成后
  3. 隐藏款调剂后
  4. 大箱盒子顺序打乱后
  5. 单盒内商品顺序打乱后
  6. 最终结果（完全随机后）

四、算法复杂度

  • 时间复杂度：O(n × m × k)，其中 n=大箱数，m=每箱盒数，k=单盒商品数
  • 空间复杂度：O(n × m)，存储所有盒子状态
  • 洗牌算法：Fisher-Yates，时间复杂度 O(n)，保证均匀随机

五、边界情况处理

  1. 库存不足：如果总库存 < 总商品数，验证3会失败
  2. 隐藏款过少：如果隐藏款总数 < 大箱数，无法通过验证1
  3. 单盒容量：如果某SKU库存 > 总盒数，会导致验证2失败（同盒重复）
  4. 调剂失败：如果没有大箱有≥2个隐藏款，但有0隐藏款的大箱，调剂无法完成

六、配置模式

简单模式

  • 自动执行完整的6步装盒策略
  • 实时展示每步结果
  • 自动进行4项验证
  • 提供可视化反馈

高级模式

  • 仅提示"原逻辑"
  • 可由开发者自定义装盒算法
