2. 需求背景与目标

2.1 现状问题与痛点

当前抽盒机产品存在以下痛点：

  1. 配置复杂：运营人员需要进行复杂配置，增加了使用难度。
  2. 缺乏简单模式：没有简单的配置模式，不利于快速上线活动。

2.2 需求提出的必要性

通过引入"简单配置模式"，可以：

  • 简化运营人员的配置流程
  • 快速实现1:1指定商品数量的配置
  • 提高活动上线的效率

2.3 业务目标

目标类型描述衡量指标目标值
配置效率提高配置效率，减少配置时间平均配置时间≤30分钟
活动上线率提高活动上线成功率活动上线成功率≥90%

3. 用户与使用场景

3.1 核心用户

  • 运营人员：负责在运营后台配置抽盒机活动。

3.2 典型场景

  1. 运营人员进入运营后台抽盒机配置菜单。
  2. 选择指定商品
  3. 展示全部SKU，并配置是隐藏款还是普通款，并指定库存数量。
  4. 指定活动箱数x盒数x单盒商品数。
  5. 选择简单模式还是高级模式（高级模式为原有逻辑，仅展示）
  6. 点击装盒完成装盒逻辑，并展示装盒结果
  7. 确认后保存活动，并将活动上线。

3.3 核心用户旅程

阶段用户触点用户行为痛点/情绪产品机会
配置运营后台配置菜单选择商品、SKU、数量复杂配置导致效率低简化配置流程

5. 详细方案

5.1 简单配置模式（admin_simple_config）

简单配置模式

功能描述：运营人员通过简单配置模式快速配置抽盒机活动。

详细逻辑
  • 展示营销工具-抽盒机活动列表功能模块，包含搜索、新增、活动列表（支持编辑、上线/下线）
  • 活动列表展示字段
    • 活动名称
    • 配置活动箱数x盒数x单盒商品数
    • 活动状态
    • 上线时间
  • 点击新增或编辑时，弹出编辑弹窗
    • 活动名称
    • 指定商品
    • 联动展示SKU，并支持配置为隐藏款或普通款，展示库存数量
    • 配置活动箱数x盒数x单盒商品数
    • 选择简单模式/高级模式
    • 点击装盒按钮，调用装盒逻辑，返回装盒结果
    • 保存活动，并返回活动详情
交互设计
  • 简洁的配置界面，突出提示需要配置的关键信息。
  • 提供一键完成装盒逻辑的按钮。
数据流向
运营人员配置商品、SKU、数量 → 系统自动完成装盒逻辑 → 活动上线

7. 异常与边界处理

  • 配置异常：配置不完整或错误，提示错误信息并引导修正。
  • 装盒逻辑异常：装盒逻辑执行失败，记录日志并通知运营人员。
