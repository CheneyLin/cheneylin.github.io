# 福袋活动后台管理系统产品需求文档

## 一、文档概述

### 1.1 文档目的

本文档面向产品经理、运营人员、测试人员，清晰描述福袋活动后台管理系统的业务逻辑、功能流程及操作规则，便于各方理解产品需求并推进落地。

### 1.2 文档读者

- 产品经理：理解功能全貌，用于需求评审
- 运营人员：了解操作规范，用于日常运营
- 测试人员：明确验收标准，用于测试用例设计

### 1.3 术语说明

| 术语 | 说明 |
|------|------|
| 福袋活动 | 用户支付固定金额参与抽奖，随机获得商品的活动 |
| 活动期数 | 一个活动可分为多期执行，每期独立计库存、独立开启 |
| 每期库存 | 单个活动期次的商品库存数量 |
| 总库存 | 每期库存 × 活动期数，即整个活动的商品总数量 |
| 申请单 | 活动上线或内容修改时，提交给审核人员的申请单据 |
| 上线状态 | 活动在用户端的展示状态：未上线 / 上线中 / 已下线 |
| 审核状态 | 活动在审核流程中的状态：待提交 / 待一审 / 待二审 / 审核通过 / 审核不通过 |

## 二、核心业务流程

### 2.1 活动生命周期

活动从创建到下线，经历以下阶段：

```
创建活动 → 编辑完善 → 提交审核 → 审核流程 → 活动上线 → 运营监控 → 活动下线
              ↑           |            |            |            |
              └───────────┴────────────┴────────────┘            |
                         （审核不通过可重新编辑提交）            |
                                                                  |
              上线后可申请修改 → 提交审核 → 审核通过 → 修改生效 ────┘
```

### 2.2 审核流程说明

#### 审核流程（3步审核）

1. 信安审核：审核活动内容合规性
2. 主管审核：审核活动配置正确性
3. 财务审核：审核成本与预算合理性
4. 审核通过后，修改内容自动生效

## 三、功能模块详解

### 3.1 活动列表页

#### 页面功能

展示所有福袋活动，支持筛选查询和快捷操作。

#### 筛选条件

| 筛选项 | 说明 |
|--------|------|
| 活动名称/ID | 输入活动名称关键字或活动ID精确查询 |
| 商品名称/ID | 输入商品名称关键字或商品ID查询关联活动 |
| 上线状态 | 全部 / 未上线 / 上线中 / 已下线 |
| 审核状态 | 全部 / 待提交 / 待一审 / 待二审 / 审核通过 / 审核不通过 |
| 信安审核 | 全部 / 待审核 / 审核通过 / 审核不通过 |

#### 列表字段

| 字段 | 说明 |
|------|------|
| ID | 活动唯一编号 |
| 活动名称 | 活动标题 |
| 活动时间 | 活动开始时间 ~ 结束时间 |
| 期数 | 活动总期数 |
| 邮寄期限 | 用户获得商品后需在N天内完成邮寄 |
| 商品 | 关联的虚拟商品ID |
| 更新时间 | 活动信息最后更新时间 |
| 状态 | 分三行展示：上线状态 / 活动审核状态 / 信安审核状态 |
| 操作 | 根据活动状态显示不同的操作按钮 |

#### 状态展示规则

状态列固定显示三行：

- 第一行：上线状态（未上线 / 上线中 / 已下线）
- 第二行：活动审核状态（待提交 / 待审核 / 审核通过 / 审核不通过）
- 第三行：信安审核状态（待审核 / 审核通过 / 审核不通过）

#### 操作按钮规则

**所有状态均支持复制功能，可快速创建新活动。**

| 活动状态 | 显示的操作按钮 |
|----------|----------------|
| 待提交 | **复制**、编辑、提交审核、查看 |
| 待一审 / 待二审 | **复制**、查看、申请单 |
| 审核通过（未上线） | **复制**、上线、编辑、查看、申请单 |
| 上线中 | **复制**、编辑、下线、抽奖订单、兑奖订单、申请单、申请修改 |
| 已下线 | **复制**、上线、抽奖订单、兑奖订单、申请单 |
| 审核不通过 | **复制**、编辑、重新提交、查看、申请单 |

#### 复制功能说明

- **所有状态的活动均可复制**
- 复制后生成新的活动草稿，状态为"待提交"
- 复制内容包括：活动名称、期数配置、邮寄截止时间、活动规则、活动图片、奖品商品池
- 复制后的活动需要重新提交审核

### 3.2 创建/编辑活动

#### 页面结构

分为6个配置区域：基础信息、期数配置、邮寄截止时间、活动展示设置、活动规则、活动图片、奖品商品池。

#### 基础信息配置

| 字段 | 是否必填 | 说明 |
|------|----------|------|
| 活动名称 | 是 | 活动标题，2-100字符 |
| 虚拟商品ID | 是 | 关联的虚拟商品，活动上线后不可修改 |
| 归属部门 | 是 | 活动所属部门，活动上线后不可修改 |
| 归属营销活动 | 否 | 关联的营销活动ID |
| 排序值 | 是 | 数值越小，活动列表排序越靠前 |

#### 期数配置

**活动期数**

- 输入范围：1-20期
- 说明：活动可分为多期执行，每期独立开启

**开启方式（二选一）**

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| A模式：售罄自动开启 | 上一期所有商品售罄后，自动开启下一期 | 希望快速售罄的活动 |
| B模式：固定时间开启 | 每期独立设置开始和结束时间 | 需要精确控制每期时间的活动 |

**B模式补充规则**

- 需为每期分别设置开始时间和结束时间
- 各期时间不能重叠（线上同一时间只能有一期活动）
- 结束时间必须晚于开始时间

#### 邮寄截止时间

- 输入天数（正整数）
- 用户获得商品后需在N天内完成邮寄，超时不可邮寄

#### 活动展示设置

- 开关：是否在活动列表页隐藏
- 开启后，活动不会出现在前端活动列表，但用户可通过直接链接访问

#### 活动规则

- 文本域输入
- 说明：活动参与规则、奖品说明、注意事项等

#### 活动图片配置

需上传4张图片：

1. 活动Banner图（尺寸：750×300像素）
2. 活动氛围图（尺寸：750×400像素）
3. 分享图标（尺寸：500×500像素）
4. 小程序分享图标（尺寸：500×400像素）

#### 奖品商品池

**功能按钮**

- 导出商品：将当前商品池导出为CSV文件
- 批量导入：通过Excel批量导入商品
- 添加奖品：从商品库选择商品添加

**商品列表字段**

| 字段 | 说明 |
|------|------|
| 商品ID | 商品唯一编号 |
| 商品信息 | 商品缩略图、名称、SKU |
| 成本价 | 商品成本价格 |
| 每期库存/总库存 | 每期库存可编辑，总库存 = 每期库存 × 期数 |
| 每期剩余/总剩余 | 当前进行中的剩余库存 / 总剩余库存 |
| 状态 | 上架中 / 已下架 |
| 操作 | 上架 / 下架 / 删除 |

**商品排序规则**

- 商品在商品池中按添加顺序展示
- 前端抽奖时商品随机打乱排序，不按照后台商品池顺序

**库存计算说明**

```
总活动库存 = 每期活动库存 × 活动期数
```

**示例：**

- 每期库存：100件
- 活动期数：3期
- 总库存：100 × 3 = 300件

#### 字段修改权限规则

| 字段类别 | 包含字段 | 创建时 | 待提交 | 上线后 |
|----------|----------|--------|--------|--------|
| 不可修改 | 虚拟商品ID、归属部门、归属营销活动 | 可编辑 | 可编辑 | 禁用，不可修改 |
| 可直接修改 | 排序值、期数、邮寄截止时间、开启方式、活动时间、展示设置 | 可编辑 | 可编辑 | 可直接修改，无需审核 |
| 需审核修改 | 活动名称、活动规则、Banner图、氛围图、分享图标、小程序分享图标 | 可编辑 | 可编辑 | 只读，需提交申请单修改 |
| 需审核修改（商品） | 奖品商品池 | 可编辑 | 可编辑 | 只读，需提交申请单修改 |

### 3.3 申请修改（上线后修改活动）

#### 功能说明

活动上线后，如需修改需审核的字段，必须通过"申请修改"功能提交申请单，审核通过后方可生效。

#### 提交条件

1. 当前没有待审核的申请单（审核中不允许提交新的申请）
2. 至少修改了一个字段（内容或商品）

#### 页面结构

分为两个折叠区域：

**区域一：内容修改（默认展开）**

- 显示当前值和新值输入框
- 支持修改：活动名称、活动规则、4张活动图片
- 图片字段显示当前图片缩略图和重新上传按钮

**区域二：商品修改（默认收起）**

- 功能按钮：导出商品、批量导入、添加奖品
- 商品列表与编辑弹窗一致
- 支持：添加新商品、修改库存、上架/下架、删除商品

#### 提交后流程

1. 生成申请单
2. 审核流程：信安审核 → 主管审核 → 财务审核
3. 审核通过后，修改内容自动生效

---

### 3.4 申请单管理

#### 功能说明

查看和管理活动的所有申请单，包括进行中的申请单和已完成的申请单。

#### 页面结构

分为两个区域：

**区域一：进行中**

- 显示待审核的申请单

**区域二：已完成**

- 显示审核通过或审核不通过的申请单

#### 申请单列表字段

| 字段 | 说明 |
|------|------|
| 申请单号 | 唯一标识，格式：AUD_YYYYMMDD_NNN |
| 提交时间 | 提交申请的时间 |
| 更新时间 | 最后更新时间 |
| 当前审核人 | 当前待审核人的账号，如security@mgtv.com |
| 审核状态 | 待信安审核 / 待主管审核 / 待财务审核 / 审核通过 / 审核不通过 |
| 操作 | 查看详情 / 审核通过 / 审核不通过 / 重新提交 |

#### 审核状态说明

| 审核状态 | 说明 |
|----------|------|
| 待信安审核 | 申请单提交后，等待信安审核 |
| 待主管审核 | 信安审核通过，等待主管审核 |
| 待财务审核 | 主管审核通过，等待财务审核 |
| 审核通过 | 财务审核通过，申请单完成 |
| 审核不通过 | 任一审核节点不通过，申请单结束 |

---

### 3.5 申请单详情

#### 页面结构

申请单详情仅显示修改内容对比表：

- 三列：字段名、修改前、修改后
- 清晰展示本次申请单的所有修改内容

## 四、核心业务规则

### 4.1 审核中操作限制

**核心原则：审核中不允许做任何修改**

| 场景 | 限制说明 |
|------|----------|
| 有待审核的申请单 | 不能提交新的申请单 |
| 有审核中的申请单 | 不能编辑活动内容、不能修改商品池 |

### 4.2 状态与操作权限

| 活动状态 | 可执行操作 | 禁止操作 |
|----------|------------|----------|
| 待提交 | **复制**、编辑、提交审核、查看 | 上线、下线、申请修改 |
| 待一审/待二审 | **复制**、查看、申请单 | 编辑、上线、下线、申请修改 |
| 审核通过（未上线） | **复制**、上线、编辑、查看、申请单 | 申请修改 |
| 上线中 | **复制**、编辑（部分字段）、下线、订单查看、申请单、申请修改 | 直接修改需审核字段、直接修改商品池 |
| 已下线 | **复制**、上线、订单查看、申请单 | 编辑、申请修改 |
| 审核不通过 | **复制**、编辑、重新提交、查看、申请单 | 上线、申请修改 |

### 4.3 审核权限

| 审核角色 | 可审核的申请单 |
|----------|----------------|
| 信安审核 | 所有申请单（一审） |
| 主管审核 | 信安审核通过后的申请单（二审） |
| 财务审核 | 主管审核通过后的申请单（三审） |

## 五、异常场景处理

### 5.1 常见异常提示

| 异常场景 | 提示信息 |
|----------|----------|
| 审核中尝试修改 | "当前有审核中的申请单，请先处理后再修改" |
| 未修改任何字段提交申请 | "请至少修改一个字段后再提交" |
| 期数时间重叠（B模式） | "第N期的时间与已有期数重叠，线上同一时间只能开启一期" |
| 添加重复商品 | "该商品已在列表中" |
| 无权限审核 | "您没有权限进行此审核操作" |

### 5.2 校验规则

**活动创建/编辑校验**

- 活动名称：必填，2-100字符
- 拟商品ID：必填
- 归属部门：必填
- 活动期数：必填，1-20
- 邮寄截止时间：必填，正整数
- 活动规则：必填，2-2000字符
- 活动图片：4张图片均需上传

**期数时间校验（B模式）**

- 每期结束时间必须晚于开始时间
- 各期时间不能重叠

## 六、附录

### 6.1 状态流转图

**活动状态流转**

```
待提交 → 提交审核 → 待一审 → 信安通过 → 待二审 → 主管通过 → 待三审 → 财务通过 → 审核通过 → 上线 → 上线中
                                                                                      |
                                                                                      下线
                                                                                      |
                                                                                      v
                                                                                  已下线

（任一节审核不通过）→ 审核不通过 → 重新编辑 → 待提交
```

**申请单状态流转**

```
提交 → 待信安审核 → 信安通过 → 待主管审核 → 主管通过 → 待财务审核 → 财务通过 → 审核通过
          |
          → 审核不通过 → 重新提交 → 待信安审核
```

### 6.2 ID编号规则

| ID类型 | 编号规则 | 示例 |
|--------|----------|------|
| 活动ID | 自增数字 | 1001, 1002, 1003 |
| 申请单号 | AUD_YYYYMMDD_NNN | AUD_20260420_001 |

### 6.3 枚举值定义

**上线状态**

- 未上线（draft）
- 上线中（online）
- 已下线（offline）

**审核状态**

- 待提交（draft）
- 待一审（first_pending）
- 待二审（second_pending）
- 审核通过（approved）
- 审核不通过（rejected）

**申请单状态**

- 待信安审核（pending_xin_an）
- 待主管审核（pending_manager）
- 待财务审核（pending_finance）
- 审核通过（approved）
- 审核不通过（rejected）

**开启方式**

- A模式：售罄自动开启
- B模式：固定时间开启
