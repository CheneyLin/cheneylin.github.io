2. 需求背景与目标

2.1 现状问题与痛点

当前抽赠商品配置流程存在以下痛点：

  1. 配置效率低：缺乏统一的抽赠商品管理工具，运营需要手动配置每个活动的赠品信息
  2. 审核流程不规范：缺少标准化的审核流程和状态管理
  3. 数据统计缺失：无法实时监控活动库存、参与人数等关键指标
  4. 费用计算复杂：不同费用策略（0元赠送、按销售价计算）需要人工核算

2.2 需求提出的必要性

通过引入"抽赠商品营销工具"，可以：

  • 提供统一的抽赠商品管理平台，提升配置效率
  • 建立标准化的审核流程，确保活动质量
  • 实时监控活动数据，支持运营决策
  • 自动化费用计算，减少人工错误

2.3 业务目标

目标类型描述衡量指标目标值
配置效率缩短抽赠商品配置时间平均配置时长≤10分钟/活动

3. 用户与使用场景

3.1 核心用户

  • 运营人员：负责抽赠商品的配置、提交和管理
  • 审核人员：负责审核抽赠商品配置的合规性
  • 数据分析师：关注活动数据和效果分析

3.2 典型场景

  1. 运营在后台创建活动玩法时，为玩法绑定虚拟商品ID并配置对应奖品商品池
  2. 运营通过输入商品ID或批量导入/Excel导入方式，将奖品商品导入商品库
  3. 导入后，运营需要调整商品 SKU 的活动库存和告警库存，并选择是否启用告警
  4. 财务审核奖品成本价与抽赠方式，确定项目的营销费用策略参数
  5. 活动类型统一使用本产品配置的奖品商品池与费用策略参数，实现通用流程管理

3.3 系统架构和流程

本节说明运营、财务、用户三类角色与三个产品模块之间的关系，明确"抽赠商品管理""抽吧唧活动管理""抽吧唧活动页面（另行规划）"三者的职责与交互边界。

5. 详细方案

5.1 运营后台端功能

5.1.1 抽赠商品管理列表页

功能描述：展示所有抽赠商品配置，支持多维度筛选和查询。

详细逻辑
  • 展示所有抽赠商品配置列表
  • 支持按项目、玩法、虚拟商品ID、状态筛选
  • 表格列包括：活动名称、玩法、虚拟商品ID、费用策略、提报人、审核状态、创建时间、操作
  • 操作按钮：编辑、查看、复制、审核（仅待审核状态显示）
  • 支持批量选择和批量操作
  • 分页展示，默认每页20条
交互设计
  • 筛选条件支持实时搜索
  • 状态徽章使用颜色区分（已通过-绿色、待审核-橙色、已拒绝-红色）
  • 悬停行高亮显示
  • 操作按钮根据状态动态显示
数据流向
前端加载页面 → 获取抽赠商品列表 → 展示数据表格
→ 用户筛选/操作 → 更新列表或跳转详情页

5.1.2 抽赠商品新增/编辑弹层

功能描述：配置抽赠商品的基本信息和商品信息。

详细逻辑
  • 基本配置区（上半区）：
    • 活动名称：文本输入框，必填
    • 玩法绑定：下拉选择，单选，默认"抽吧唧活动"
    • 虚拟商品ID：多选搜索框，支持标签式添加，支持多个虚拟商品ID
    • 费用策略：单选按钮组，默认"0元赠送-按销售价计算营销费用"
      • 0元赠送-按销售价计算营销费用（默认项目）
      • 关联虚拟订单合并计算营销费用
    • 告警设置：开关按钮，默认关闭，开启后可设置告警库存
  • 商品信息区（下半区）：
    • 商品列表表格：商品ID、SKU、商品名称、成本价、活动库存、告警库存、告警状态
    • 支持批量设置库存和告警库存
    • 支持导入商品（Excel上传）
    • 支持删除商品
  • 保存时进行表单验证，必填项不能为空
  • 保存成功后返回列表页并提示成功
交互设计
  • 弹层分为上下两个区域，清晰区分基本配置和商品信息
  • 虚拟商品ID支持回车快速添加标签
  • 商品表格支持滚动查看
  • 保存按钮固定在底部
数据流向
用户打开弹层 → 填写配置信息 → 添加商品
→ 表单验证 → 保存数据 → 返回列表页

5.1.3 抽赠商品审核

功能描述：审核人员对待审核的抽赠商品配置进行审核操作。

详细逻辑
  • 审核人员在列表页点击"审核"按钮
  • 弹出审核对话框，展示配置详情
  • 审核人员可以选择"通过"或"拒绝"
  • 拒绝时需要填写拒绝原因
  • 审核通过后，配置生效并可关联到活动
  • 审核拒绝后，提报人可以修改后重新提交
交互设计
  • 审核对话框清晰展示配置详情
  • 通过和拒绝按钮使用主次按钮区分
  • 拒绝原因输入框在点击"拒绝"后展开
数据流向
审核人员点击审核 → 查看详情 → 选择通过/拒绝
→ 更新审核状态 → 通知提报人

5.1.4 抽吧唧活动管理列表页

功能描述：展示所有抽吧唧活动，支持筛选和状态管理。

详细逻辑
  • 展示所有抽吧唧活动列表
  • 支持按活动名称、虚拟商品ID、状态筛选
  • 表格列包括：活动名称、虚拟商品ID、活动状态、开始时间、结束时间、操作
  • 操作按钮：编辑、启用/禁用、查看
  • 支持展开查看详情（库存统计、参与人数、发放数量）
  • 分页展示，默认每页20条
交互设计
  • 状态徽章使用颜色区分（进行中-绿色、未开始-橙色、已结束-灰色）
  • 展开图标点击后旋转90度
  • 详情行展示库存和统计数据
  • 已结束的活动不可编辑和禁用
数据流向
前端加载页面 → 获取活动列表 → 展示数据表格
→ 用户筛选/展开/操作 → 更新列表或跳转详情页

5.1.5 活动详情查看

功能描述：查看活动的完整配置和详细数据统计。

详细逻辑
  • 展示活动基本信息（名称、ID、时间、状态）
  • 展示关联的抽赠商品配置
  • 展示库存统计（总库存、已使用、剩余）
  • 展示参与数据（参与人数、发放数量）
  • 展示费用统计（营销费用、实际支出）
交互设计
  • 使用卡片式布局展示不同维度的数据
  • 关键数据使用大字号突出显示
  • 支持导出详细数据报表
数据流向
用户点击查看详情 → 加载活动详情 → 展示配置和数据

5.1.6 活动启用/禁用

功能描述：控制活动的启用和禁用状态。

详细逻辑
  • 运营可以在列表页点击"启用"或"禁用"按钮
  • 启用活动后，用户可以参与活动
  • 禁用活动后，用户无法继续参与，但已参与的数据保留
  • 已结束的活动不可再次启用
  • 操作前需要二次确认
交互设计
  • 启用和禁用按钮根据当前状态动态显示
  • 点击后弹出确认对话框
  • 操作成功后刷新列表状态
数据流向
用户点击启用/禁用 → 二次确认 → 更新活动状态
→ 刷新列表 → 通知相关人员

7. 异常与边界处理

  • 网络异常：保存或提交时网络中断，保留草稿，网络恢复后提示继续提交
  • 库存不足：活动库存小于已发放数量时，禁止禁用操作并提示
  • 并发审核：同一配置被多人同时审核时，数据库层面加锁，确保原子性
  • 数据一致性：活动状态变更时，同步更新关联的抽赠商品状态
  • 空状态处理：
    • 抽赠商品列表（无数据）：展示"暂无抽赠商品配置，点击新增开始配置"
    • 活动列表（无数据）：展示"暂无抽吧唧活动，请先配置抽赠商品"
  • 权限限制：非审核人员不显示审核按钮，非运营人员不显示编辑按钮
  • 重复提交：防止用户多次点击提交按钮，提交后按钮置灰
