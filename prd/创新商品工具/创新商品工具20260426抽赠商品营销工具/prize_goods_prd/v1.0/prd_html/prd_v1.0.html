2. 需求背景与目标

2.1 现状问题与痛点

当前抽赠商品配置流程存在以下痛点：

  1. 配置效率低：缺乏统一的抽赠商品管理工具，运营需要手动配置每个活动的赠品信息
  2. 审核流程不规范：缺少标准化的审核流程和状态管理
  3. 数据统计缺失：无法实时监控活动库存、参与人数等关键指标
  4. 费用计算复杂：不同费用策略（0元赠送、按销售价计算）需要人工核算

2.2 需求提出的必要性

通过引入"抽赠商品营销工具"，可以：

  • 提供统一的抽赠商品管理平台，提升配置效率
  • 建立标准化的审核流程，确保活动质量
  • 实时监控活动数据，支持运营决策
  • 自动化费用计算，减少人工错误

2.3 业务目标

目标类型描述衡量指标目标值
配置效率缩短抽赠商品配置时间平均配置时长≤10分钟/活动
审核效率提升审核流程速度平均审核时长≤2小时
数据准确性确保库存和费用计算准确数据准确率≥99.9%
用户满意度提升运营使用体验NPS评分≥8分

2.4 用户目标

目标类型描述衡量指标
操作便捷性运营能够快速配置和管理抽赠商品任务完成率
数据可视化清晰展示活动数据和库存状态数据查询效率
流程规范性标准化的审核和提交流程流程合规率

3. 用户与使用场景

3.1 核心用户

  • 运营人员：负责抽赠商品的配置、提交和管理
  • 审核人员：负责审核抽赠商品配置的合规性
  • 数据分析师：关注活动数据和效果分析

3.2 典型场景

  1. 运营在后台创建活动玩法时，为玩法绑定虚拟商品ID并配置对应奖品商品池
  2. 运营通过输入商品ID或批量导入/Excel导入方式，将奖品商品导入商品库
  3. 导入后，运营需要调整商品 SKU 的活动库存和告警库存，并选择是否启用告警
  4. 财务审核奖品成本价与抽赠方式，确定项目的营销费用策略参数
  5. 活动类型统一使用本产品配置的奖品商品池与费用策略参数，实现通用流程管理

3.3 系统架构和流程

本节说明运营、财务、用户三类角色与三个产品模块之间的关系，明确"抽赠商品管理""抽吧唧活动管理""抽吧唧活动页面（另行规划）"三者的职责与交互边界。

5. 详细方案

5.1 运营后台端功能

5.1.1 运营后台-营销管理-抽赠商品管理模块

5.1.1.1 抽赠商品管理列表页

入口位置：运营后台 > 营销管理 > 抽赠商品管理

功能描述：展示所有抽赠商品配置，支持多维度筛选和查询。

页面模块
  • 抽赠商品列表展示：活动名称、玩法、费用策略、归属部门、归属营销活动、提报人、活动时间、活动审核、商品审核、操作等关键信息
    • 活动时间：上面是开始时间，下面是结束时间，时分秒级展示
    • 商品审核：展示已上线商品数量，待审核商品数量
  • 操作按钮：
    • 单独的新增按钮
    • 列表中的编辑、递交审核/撤回审核/审核/启用/停用、复制
  • 筛选条件：项目、玩法、状态
交互设计
  • 筛选条件支持实时搜索
  • 状态徽章使用颜色区分（已通过-绿色、待审核-橙色、已拒绝-红色）
  • 悬停行高亮显示
  • 操作按钮根据状态动态显示
  • 分页展示，默认每页20条
数据流向
前端加载页面 → 获取抽赠商品列表 → 展示数据表格
→ 用户筛选/操作 → 更新列表或跳转详情页

5.1.1.2 抽赠商品管理添加弹层

入口位置：运营后台 > 营销管理 > 抽赠商品管理 - 新增

目标：支持运营完成玩法绑定与项目级费用、告警的基础配置

页面模块
  • 基本配置（上半区）：
    • 活动名称：输入活动名称
    • 玩法绑定：选择活动玩法，单选
      • 默认项目：抽吧唧活动
      • 抽福袋活动
    • 虚拟商品ID：支持多个虚拟商品ID，支持输入ID下拉搜索选择框添加
    • 费用策略：选择活动使用的策略，单选
      • 0元赠送-按销售价计算营销费用（默认项目）
      • 关联虚拟订单合并计算营销费用
    • 归属部门：选择活动归属部门，单选
    • 归属营销活动：选择活动归属营销活动，单选
    • 活动起止时间：选择活动开始时间和结束时间
    • 可叠加营销工具：选择活动可叠加的营销工具，多选
      • 优惠券
      • 支付有礼
    • 告警设置：是否启用告警开关，默认关闭
  • 商品信息（下半区）：
    • 商品列表：支持输入商品ID下拉搜索选择框添加，展示已添加的商品ID、SKU、商品名称、成本价、活动库存、告警库存、告警状态等信息
    • 批量操作：批量设置活动库存和告警库存，批量启用/关闭告警
    • 导入操作：支持批量导入/Excel导入方式添加商品（本方案中仅示意）
  • 保存与取消按钮
交互设计
  • 弹层分为上下两个区域，清晰区分基本配置和商品信息
  • 虚拟商品ID支持回车快速添加标签
  • 商品表格支持滚动查看
  • 保存按钮固定在底部
  • 表单验证：必填项不能为空
数据流向
用户打开弹层 → 填写配置信息 → 添加商品
→ 表单验证 → 保存数据 → 返回列表页

7. 异常与边界处理

  • 网络异常：保存或提交时网络中断，保留草稿，网络恢复后提示继续提交
  • 库存不足：活动库存小于已发放数量时，禁止禁用操作并提示
  • 并发审核：同一配置被多人同时审核时，数据库层面加锁，确保原子性
  • 数据一致性：活动状态变更时，同步更新关联的抽赠商品状态
  • 空状态处理：
    • 抽赠商品列表（无数据）：展示"暂无抽赠商品配置，点击新增开始配置"
    • 活动列表（无数据）：展示"暂无抽吧唧活动，请先配置抽赠商品"
  • 权限限制：非审核人员不显示审核按钮，非运营人员不显示编辑按钮
  • 重复提交：防止用户多次点击提交按钮，提交后按钮置灰
