2. 需求背景与目标

2.1 现状问题与痛点

目前"电子吧唧"设备仅支持导入平台已授权的官方图片内容。用户拥有大量个人相册中的珍贵图片（如自拍、偶像照片、宠物等），却无法将其转化为设备上的虚拟徽章，导致用户使用场景单一，设备利用率低。

2.2 需求提出的必要性

  • 满足用户个性化展示的需求，提升用户粘性
  • 考虑到版权与信息安全，该功能定位为"纯传输工具"，不提供社区分享，仅作为用户个人的私人素材库扩充
  • 现有流程无法满足用户导入个人图片的需求

2.3 业务目标

目标类型描述衡量指标目标值
核心业务支持用户从手机相册选择图片并导入设备导入功能使用率上线首月 DAU 渗透率达到 15%
用户体验简化导入流程，确保传输稳定性导入成功率成功率 > 95%
安全合规确保导入内容符合法律法规协议签署率100% 签署安全协议

5. 详细方案

5.1 导入入口 feature_import_entry

功能描述：在"吧唧背包"页面顶部增加"导入图片"按钮，用户点击后进入图片导入流程。

详细逻辑
  • 位置："吧唧背包"页面顶部，设备名称/昵称区域的右侧
  • 文字："导入图片"
  • 交互：点击按钮唤起系统相册选择器
  • 前置条件：设备处于"已连接"状态。若未连接，按钮置灰，点击提示"请先连接设备"
数据流向
用户点击导入按钮 → 检查设备连接状态
→ 若已连接，检查安全协议状态 → 进入下一步

5.2 安全协议弹窗 feature_security_agreement

功能描述：首次使用时弹出安全协议弹窗，用户需勾选同意后方可继续。

详细逻辑
  • 触发逻辑：检测本地存储标志位 `has_agreed_security`
  • 若为 `false`，点击入口时弹出模态对话框
  • 用户勾选"我已阅读并同意《信息安全约定》"复选框后，"确认导入"按钮高亮可点
  • 确认后保存标志位为 `true`，后续不再弹窗
  • 协议文案："我承诺上传的图片不包含任何违规、侵权及不良信息，仅作为个人使用。"
交互设计
  • 模态框居中显示，背景半透明遮罩
  • 协议内容清晰可读，勾选框明显
  • 未勾选时"确认导入"按钮置灰，不可点击

5.3 图片选择与裁剪 feature_image_crop

功能描述：用户从系统相册选择图片后，进入裁剪界面，支持正方形裁剪。

详细逻辑
  • 选择器：支持单选模式
  • 裁剪逻辑：进入裁剪页，默认显示正方形裁剪框
  • 不支持旋转、滤镜等其他操作
  • 点击"确认"后，系统在本地进行压缩处理（沿用现有压缩算法）
交互设计
  • 裁剪框固定正方形比例，角标可拖动调整
  • 背景图片可缩放、移动
  • 底部显示"返回"和"确认导入"按钮

5.4 传输逻辑 feature_transfer_logic

功能描述：图片裁剪确认后，自动开始传输到设备，传输完成即视为功能结束。

详细逻辑
  • 裁剪确认后，自动开始传输
  • 显示"正在导入..."的 Loading 状态
  • 传输成功后，Toast 提示"导入成功"，2 秒后自动关闭弹窗/页面，返回背包主页
  • 注意：导入的图片不显示在"已拥有"列表中，仅作为设备素材存在
数据流向
用户确认裁剪 → 本地压缩图片
→ 蓝牙/Wi-Fi 传输到设备 → 设备接收完成
→ Toast 提示成功 → 2 秒后返回背包主页

5.5 异常处理 feature_error_handling

功能描述：处理传输过程中可能出现的各种异常情况。

详细逻辑
  • 断连/失败：若传输过程中设备断开或数据校验失败，立即停止传输
  • Toast 提示文案："传输失败，请重试"
  • 自动恢复：Toast 显示 2 秒后，自动关闭当前操作流，返回"吧唧背包"主界面，避免用户停留在不可操作的中间态
其他异常场景
  • 权限拒绝：若用户拒绝相册权限，引导用户前往系统设置开启
  • 图片格式不支持：仅支持 JPG/PNG 格式，若选择其他格式（如 HEIC, GIF），提示"不支持的图片格式"
  • 存储空间不足：若设备存储空间已满，提示"设备空间不足，请清理后再试"

6. 业务流程图

flowchart TD Start([用户点击导入图片]) --> CheckConnection{设备是否已连接?} CheckConnection -->|否| Toast1[Toast: 请先连接设备] Toast1 --> End1([结束]) CheckConnection -->|是| CheckProtocol{是否首次使用?} CheckProtocol -->|是| ShowProtocol[显示安全协议弹窗] ShowProtocol --> UserAgree{用户是否勾选同意?} UserAgree -->|否| End2([取消导入]) UserAgree -->|是| SaveProtocol[保存协议状态到本地] SaveProtocol --> SelectImage CheckProtocol -->|否| SelectImage[选择手机相册图片] SelectImage --> CropImage[进入裁剪页] CropImage --> ConfirmCrop{用户是否确认裁剪?} ConfirmCrop -->|否| End3([返回背包页]) ConfirmCrop -->|是| CompressImage[本地压缩图片] CompressImage --> StartTransfer[开始传输到设备] StartTransfer --> TransferResult{传输是否成功?} TransferResult -->|成功| ToastSuccess[Toast: 导入成功] ToastSuccess --> Wait2s[等待 2 秒] Wait2s --> End4([自动返回背包主页]) TransferResult -->|失败| ToastFail[Toast: 传输失败，请重试] ToastFail --> Wait2sFail[等待 2 秒] Wait2sFail --> End5([自动返回背包主页])

7. 异常与边界处理

  • 权限拒绝：若用户拒绝相册权限，引导用户前往系统设置开启
  • 图片格式不支持：仅支持 JPG/PNG 格式，若选择其他格式（如 HEIC, GIF），提示"不支持的图片格式"
  • 存储空间不足：若设备存储空间已满，提示"设备空间不足，请清理后再试"
  • 网络异常：传输过程中网络中断，提示"网络不稳定，请检查网络后重试"
  • 设备断开：传输过程中设备断开连接，立即停止传输并提示"设备已断开，请重新连接"
